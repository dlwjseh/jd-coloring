- **한 줄 요약**: (추후 보강) 온 가족/여러 사용자가 각자 프로필로 즐기는 컬러링 앱

## 화면 1: 사용자 선택 (User Selection) — P0

### 목적
앱 실행 시 가장 먼저 나타나는 화면. 사용할 프로필을 고르거나 새로 만든다. (넷플릭스·iPad 프로필 선택과 유사)

### 레이아웃
- 등록된 사용자가 **원형 프로필**로 **가로 한 줄** 배치
- **정렬**: 기본은 **가운데 정렬**(프로필이 화면 폭 안에 다 들어오면 가로 중앙으로 모음). **화면을 넘치면 좌측 정렬 + 좌우 드래그(수평 스크롤)** 로 전환
- 각 프로필: 원형 이미지 + 그 아래 **이름** 라벨
- 화면 **구석에 '사용자 추가' 버튼** 배치 (위치는 디자인에서 확정 — 우상단 모서리 또는 프로필 줄 끝의 '+' 원 제안)
- **설정 버튼** — 화면 한켠 코너에 배치 (SF Symbol `gearshape`). 탭하면 **앱 설정 시트**가 열린다. 위치는 '사용자 추가' 버튼과 반대쪽 코너로 디자인에서 확정. *(추가 2026-06-04)*

### 사용자 흐름 (User Flow)
```
앱 실행
  ├─ 등록된 사용자 있음 → 프로필 가로 목록 표시
  │     ├─ 프로필 탭        → 다음 화면 진입  ※ 진입 후 동작은 다음 단계에서 기획
  │     └─ 프로필 롱프레스  → [수정] / [삭제] 메뉴
  │            ├─ 수정 → 사용자 추가 화면 재사용(이름·이미지 변경)
  │            └─ 삭제 → 확인 다이얼로그 → 확인 시 삭제
  └─ 등록된 사용자 없음(첫 실행) → 곧바로 '사용자 추가' 화면
```

### 기능 상세

**1) 사용자 추가 — P0**
- 진입: 추가 버튼 탭, 또는 첫 실행(사용자 0명) 시 자동
- 입력: **이름**(텍스트) + **프로필 이미지**
- 이미지 소스: **사진 앨범/갤러리에서 선택** (카메라·기본 아바타는 이번 범위 밖)
  - iOS/iPadOS 사진 라이브러리 접근 권한 처리 필요
  - 원형 표시에 맞춰 **정사각/원형 크롭** 단계 필요
- 저장 시 프로필 목록에 새 원형으로 추가

**2) 사용자 수정 — P0**
- 진입: 프로필 롱프레스 → [수정]
- 추가 화면을 재사용해 이름·이미지를 변경 후 저장

**3) 사용자 삭제 — P0**
- 진입: 프로필 롱프레스 → [삭제]
- **확인 다이얼로그** 표시("정말 삭제할까요?") → 확인 시에만 삭제
- (해당 사용자의 작업물 처리 정책은 작업물 기능 기획 시 결정 — 열린 질문)

**4) 프로필 선택(진입) — 다음 단계**
- 프로필 탭 시 다음 화면으로 이동. 이동 후 화면/동작은 **별도 기획 예정**

### 화면 전환 / 애니메이션 — P1

전부 UI 레벨 연출이므로 **순수 SwiftUI로 구현**(Metal 불필요). 느낌은 빠르고 경쾌하게("샤악/스윽"), 끝에 살짝 스프링으로 정착.

**A1. 화면 진입 — 프로필 등장**
- **트리거**: 사용자 선택 화면에 올 때마다 (앱 실행 + 다른 화면에서 복귀 시 매번 재생)
- **동작**: 프로필 원들이 화면 **오른쪽 바깥 → 제자리로 슬라이드 인** (오른쪽에서 왼쪽 방향), 페이드 인 동반
- **방식**: **순차(stagger)** — 각 프로필이 약간의 시차(제안: 50~80ms 간격)를 두고 하나씩 "주르르르" 등장
- 끝에 가벼운 스프링 정착감

**A2. 등록·수정 진입 — 프로필 흩어짐 + 폼 등장**
- **트리거**: '추가' 버튼 탭, 또는 롱프레스 → [수정]
- **1단계**: 현재 프로필들이 **좌·우로 갈라져 화면 밖으로 흩어짐**(가운데를 비움)
- **2단계**: 비워진 가운데로 **수정 UI(이름 입력칸 + 이미지 선택)가 스윽 등장** — 위/아래 방향은 디자이너 결정(제안: 아래에서 위로)
- 두 단계는 살짝 겹쳐(overlap) 자연스럽게 이어짐
- **역방향(reverse)**: 저장/취소로 빠져나갈 때 폼이 사라지고 프로필이 **다시 가운데로 모이며** 복귀

**구현 메모(개발자용)**
- SwiftUI `transition` + `withAnimation`(spring), stagger는 항목 인덱스 기반 delay로 구현
- `matchedGeometryEffect` / `phaseAnimator` 활용 검토
- ⚠️ **평가자 체크포인트**: 타이머 남발로 stagger 구현 금지, 프로필 多일 때 동시 애니메이션 부하, "매 진입 재생"이라 무거우면 바로 체감됨 → 가볍게 유지

### 데이터/저장 요구사항
- 프로필 = { id, 이름, 이미지 } 를 **기기 로컬에 영구 저장**
- 선택한 갤러리 이미지를 앱 내부 저장소에 복사·보관(원본 삭제돼도 유지)
- 프로필 표시 순서(추가 순/최근 사용 등)는 추후 결정 — 열린 질문

### 플랫폼 고려사항
- **iPad 전용**: 터치로 탭·롱프레스·수평 드래그. 사진 앨범 접근.
- **Mac 지원 종료(2026-06-04)**: Mac 대상 분기(`#if os(macOS)`) 및 Mac 전용 동작 코드를 제거. macOS 타깃 자체를 프로젝트에서 삭제.

### 비기능 요구사항
- 수평 스크롤·드래그가 부드럽게(60fps) 동작
- 프로필 이미지 다수여도 썸네일 다운샘플링으로 메모리 절약

### 범위 밖 (Out of Scope) — 이번 화면에서 안 함
- 카메라 촬영 / 기본 제공 아바타
- 어른 잠금·비밀번호 등 강한 보호장치(삭제 확인 다이얼로그까지만)
- 프로필 선택 이후의 화면들
- 설정 화면의 구체 UI → 아래 §앱 설정 화면 참조

### 열린 질문 (Open Questions)
1. 프로필 **최대 개수** 제한이 있는가? (기본: 제한 없음, 스크롤로 처리)
2. 프로필 **정렬 순서** 기준은? (추가순 / 최근 사용순)
3. 사용자 **삭제 시 그 사용자의 작업물**도 함께 삭제하는가?
4. '사용자 추가' 버튼의 정확한 위치/형태 (→ 디자인에서 확정)

## 화면 2: 색칠 도안 갤러리 (Coloring Gallery) — P0

### 목적
프로필 선택 후 진입하는 화면. **색칠할 도안들이 그리드로 펼쳐지고**, 칸을 탭하면 색칠을 시작한다. 새 도안을 **구석의 추가 버튼**으로 업로드한다. (사진첩 앱의 격자 + 컬러링 보관함 느낌)

### 핵심 결정 (이번 세션 확정)
1. **도안은 공유 · 작업물은 개인** — 누군가 올린 도안은 온 가족(모든 프로필)이 함께 보고 각자 색칠. 색칠 결과(작업물)는 프로필별로 따로 저장.
2. **그리드 칸 = 현재 프로필의 진행 상태** — 내가 칠하던 도안은 칠한 모습, 안 건드린 도안은 원본(빈) 도안 썸네일.
3. **업로드 = 사진 그대로 도안 등록** — 사진을 골라 미리보기로 확인한 뒤 가공 없이 도안으로 추가. (라인아트 자동 변환은 결과 품질이 들쭉날쭉해 **제거** — 2026-06-01 결정)

### 데이터 모델 (개발자 인계)
- **도안(Template)** — 전역 공유: `{ id, name: String?(옵션), 도안 이미지(영구저장), 추가일, 추가한 프로필 id(옵션) }` (변환 제거로 '원본 이미지' 별도 보관 필드 불필요)
- **작업물(Artwork)** — (프로필 × 도안)당 최대 1개: `{ id, 프로필 id, 도안 id, 색칠 진행 데이터, 진행 썸네일, 마지막 수정일 }`
  - 색칠 "진행 데이터"의 구체 형식은 **화면 3에서 `PKDrawing` data로 확정**(`Artwork.progressData`). 이번(갤러리) 화면은 *썸네일만* 표시.
- 그리드 셀 표시 규칙: 해당 도안에 현재 프로필의 작업물이 있으면 그 진행 썸네일, 없으면 빈 도안 썸네일.

### 레이아웃
- **상단 바**: 현재 프로필(작은 원 + 이름) + **뒤로(프로필 선택으로)**. "○○의 색칠 도안" 정도의 타이틀.
- **본문**: 적응형 그리드(`LazyVGrid`, 화면 폭에 따라 열 수 자동). 각 셀은 **라운드 카드**(도안 썸네일 + 이름(있을 때만) + 부드러운 그림자), 화면 1과 같은 크림/파스텔 톤. 이름이 있으면 썸네일 아래에 한 줄 라벨로 표시; 없으면 라벨 영역을 표시하지 않는다(카드 높이 변동 없게 고정 또는 nil 처리는 디자인에서 결정).
- **구석 추가 버튼**: 화면 1의 추가 버튼과 동일 톤·위치(우하단 코너 플로팅 원, "도안 추가").
- **빈 상태(도안 0개)**: 안내 문구 + 추가 유도(첫 진입 시 곧바로 업로드로 유도할지 디자인에서 결정).

### 사용자 흐름
```
프로필 선택 → 갤러리 진입
  ├─ 도안 있음 → 그리드 표시
  │     ├─ 셀 탭         → 색칠 캔버스 진입  ※ 캔버스는 다음 단계 기획
  │     ├─ 셀 롱프레스   → [이름 수정] / [내 색칠 초기화] / [도안 삭제] (※ 공유 삭제 정책 주의 — 열린 질문)
  │     └─ 추가 버튼 탭  → 사진 선택 → 라인아트 변환 미리보기 → 저장
  └─ 도안 없음 → 빈 상태(추가 유도)
  └─ 뒤로 → 화면 1(프로필 선택)
```

### 도안 업로드 — P0
- 사진 선택: `PhotosPicker`(화면 1과 동일 경로).
- 선택하면 **'도안 만들기' 시트에 크게 미리보기** → [취소] / [도안 저장]으로 확인 후 추가. (사진 1장 = 도안 1개)
- **도안 이름 입력 필드**: 미리보기 아래에 텍스트 필드("도안 이름(선택)") 제공. **입력은 선택 사항** — 비워두고 저장해도 무방.
- **라인아트 자동 변환 제거(2026-06-01)**: 사진을 가공 없이 그대로 도안으로 등록한다. 사용자가 이미 선이 있는 색칠 도안 이미지를 올린다고 가정.
- ⚠️ **평가자 체크포인트**: 업로드 이미지는 다운샘플해 보관. **그리드 표시용 썸네일**과 (색칠용) 표시 이미지의 해상도를 분리(검수 increment4 지적). 디코딩·IO는 백그라운드.

### 화면 전환 / 애니메이션 — P1

**G1. 프로필 ↔ 갤러리 전환 — 흩어짐 + 밑에서 올라옴** *(확정 2026-06-02, 디자인 스펙 §12 / 목업 `16-anim-G1-slide.svg`)*
- **트리거**: 프로필 선택 화면에서 프로필 원을 **탭** → 갤러리 진입. '뒤로' → 복귀.
- **정방향 동작**: ① 프로필 화면 요소가 **빠짐**(타이틀 위로, 프로필 원들 좌·우로 흩어짐, 추가 버튼 아래로) → ② 갤러리 **도안 카드가 밑에서 우수수 올라옴**(stagger) + 타이틀·버튼 등장. 화면 전환 자체는 **무애니**(가로 슬라이드 제거), 두 화면이 같은 배경을 공유해 매끄럽게 이어짐.
- **역방향(reverse)**: 정방향의 **대칭 역재생** — 갤러리 빠짐(타이틀 위로, 카드·버튼 아래로) → 프로필 복귀(흩어졌던 것이 제자리로).
- **배포 타깃**: **iOS 18 이상**(확정, macOS 지원 종료 2026-06-04).
  - ※ 초기엔 네이티브 zoom 전환(`.navigationTransition(.zoom)`)으로 잡았으나 펼침 느낌이 어색해 위 방식으로 변경(2026-06-02). iOS 18 타깃은 유지.
- ⚠️ **평가자 체크포인트(반영 완료)**: stagger는 타이머 금지(`.animation().delay()`), 전환 지연은 취소 가능한 `Task.sleep`, 연타/재진입 가드(중복 push·dangling dismiss 차단), **Reduce Motion 대응**(이동·stagger 생략), 도안 多 시 60fps 유지.

### 범위 밖 (이번 화면에서 안 함)
- 실제 색칠 캔버스(붓·채우기·되돌리기 등) — 별도 화면 기획.
- 도안 카테고리/검색/정렬 옵션 UI(초기엔 단순 최신순).
- ~~라인아트 변환~~ 기능 **제거**(2026-06-01). 추후 필요 시 별도 기획.

### 열린 질문
1. **공유 도안 삭제 권한** — 아무 프로필이나 삭제 가능? 올린 사람만? 삭제 시 다른 사람 작업물도 사라지므로 확인 다이얼로그 강화 필요.

### 채색 도구 — 마커 + 색연필 — P1 추가 (2026-06-02)

> 배경: 현재 채색은 영역(칸)을 잠그고 그 칸 픽셀을 **완전 불투명(alpha 255) 단색**으로 채운다(`RegionPaintEngine.stamp`). 납작한 면이라 결·농담이 없다. "좀 더 색연필 질감"을 원한다는 요청 → **색연필 도구를 별도로 추가**하고 기존 단색 채움은 '마커'로 유지해 둘을 전환한다.

**확정 (이번 세션)**
1. **도구는 두 가지, 전환식** — 기존 단색 채움 = **마커**(또는 '채우기' 톤, 명칭은 디자인 재량), 신규 = **색연필**. 우측 레일의 도구 전환 컨트롤로 선택. 둘 다 **영역 가두기(칸 안에서만 칠해짐)·굵기 3단계·지우개**는 동일하게 적용.
2. **색연필 핵심 동작 = 쌓임(build-up) + 종이결 입자.**
   - **쌓임(반투명 누적)**: 한 번 칠하면 옅게, 같은 칸을 **여러 획으로 덧칠하면 점점 진해진다**(최대 = 그 색의 진한 값에서 멈춤). 가장 색연필다운 특성.
   - **종이결/거친 입자**: 칠한 면에 미세한 종이 질감 입자가 비쳐, 균일한 면이 아니라 까끌한 결이 보인다.
3. **세기 = 은은하게** — 멀리서 보면 거의 채워 보이고, 가까이서 보면 살짝 결·농담이 느껴지는 수준. 과한 노이즈/얼룩짐 금지.

**색연필 동작 상세 (개발 인계)**
- **반투명 스탬프**: 마커처럼 alpha를 255로 덮어쓰지 말고, 색을 **부분 불투명도로 위에 올려(source-over 알파 합성)** 칠할수록 누적되게 한다. (옅은 1패스 → 덧칠로 진해짐.)
- **한 획 안에서는 균일, 획을 새로 그으면 진해짐**: 하나의 연속 드래그(다운→업) 동안에는 같은 픽셀이 겹쳐도 **이중 적용되지 않게** 한다(획 내 커버리지 마스크로 1회만 반영). 안 그러면 스탬프가 촘촘히 겹치는 곳만 얼룩진다. 새 획을 그어 그 위에 또 칠하면 그때 진해진다.
- **종이결 입자**: 미리 만든(또는 작은 타일) **그레인 노이즈 마스크**로 픽셀별 불투명도를 살짝 변조. '은은하게'이므로 변조 폭은 작게(예: 불투명도의 ±소량). 매 스탬프마다 노이즈를 새로 생성하지 말고 **고정 텍스처를 화면 좌표에 고정해 룩업**(움직여도 결이 떨림 없이 종이에 고정된 느낌).
- **영역 경계는 그대로 보존**: 경계(라벨 0) 픽셀은 색연필도 절대 안 칠함 → 검은 선 유지.
- **지우개**: 색연필 칠도 지워야 하므로 기존 지우개(해당 칸 픽셀 alpha 0)로 동일 처리.
- **저장/썸네일**: 반투명 누적 결과가 색칠 버퍼 픽셀에 그대로 반영되므로 기존 저장 경로(`progressData` PNG + 합성 썸네일) 변경 없음.

**⚠️ 평가자 체크포인트 (색연필)**
- 스탬프 픽셀 루프에 그레인 룩업 + 알파 합성이 추가됨 → **마커보다 픽셀당 비용 증가**. 60fps 유지 확인(특히 큰 굵기·넓은 칸).
- 그레인 텍스처는 **1회 생성 후 재사용**(매 프레임/매 스탬프 재생성 금지).
- "획 내 1회 반영" 커버리지 마스크가 칸 크기만큼 매번 새 버퍼를 alloc하지 않게(재사용/부분 영역만).

**범위 밖 / 추후 (색연필)**

### 색연필 필압 반응 (Pressure → 농담) — 구현 완료 (2026-06-05)

> 배경: 색연필은 반투명 쌓임(첫칠/덧칠 2단계) + 종이결까지 들어왔지만 칠하는 **세기(필압)** 는 안 받았다. Apple Pencil의 `force`를 받아 **살짝→옅게, 꾹→진하게** 하여 펜 색칠 경험을 완성한다. 마침 입력 레이어가 이미 `PenOnlyGestureView`(UIKit 터치 + coalesced + penOnly + 줌/팬)로 정비돼 있어, 그 위에 필압을 얹었다.

**확정·구현**
1. **적용 도구 = 색연필만.** 브러쉬(마커, 단색 alpha 255 꽉 채움)·지우개는 **필압 무관**(동작 그대로). 마커는 단색 꽉 채움이 정체성이라 농담을 넣지 않는다.
2. **필압이 바꾸는 것 = 농담(진하기)뿐.** 굵기는 레일의 4단계 그대로. 살짝 누르면 한 획이 옅게, 꾹 누르면 진하게.
3. **두 메커니즘 공존** — 필압(한 획 안의 세기) + 기존 쌓임(첫칠 진하게 → 덧칠로 더 진해짐)이 **직교하게 함께** 작동. 필압은 "이번 스탬프의 한획 알파 배율"만 정하고, 덧칠 게이트/첫칠·덧칠 LUT 분기는 그대로.
4. **세기 = 은은하게(색연필 톤 유지)** — 필압 배율 상한 = **1.0(=기본 첫칠/덧칠 알파)**. 더 진하게는 **덧칠로만**. 바닥값 `minScale=0.35`로 아주 살짝 닿아도 보이게. 매핑 `scale = minScale + (1-minScale)·pow(p, gamma)`(`gamma=1.0`, 실기기 튜닝 여지).
5. **손가락 = 기본 세기** — `UITouch.type != .pencil` 또는 `maximumPossibleForce==0`(USB-C 펜슬 등)이면 pressure=nil → 배율 1.0(농담 변화 없음, 기존 동작). 펜 전용 모드(penOnly)와 무관하게 안전.

**구현 위치**
- `DrawingCanvas.swift` `PenOnlyGestureView.Inner`: `pressure(of:)`(force/maxForce, 펜 아니면 nil) 추가. `touchesBegan`/`touchesMoved`(coalesced 각 샘플)에서 위치+필압을 `onChanged(CGPoint, CGFloat?)`로 전달.
- `RegionPaintEngine.swift`: `strokeChanged(at:viewSize:pressure:)` 추가(기본값 nil → 무필압 호출 호환). `alphaScale(forPressure:)` 매핑, `stampLine`이 구간에서 배율 선형 보간, `stamp(...,alphaScale:)`이 배율을 **0~256 고정소수점(scaleFx)으로 1회 변환**해 픽셀 알파에 `(base*scaleFx+128)>>8` 적용. **scaleFx=256이면 결과가 base와 비트 동일** → 손가락·마커·지우개 회귀 0.

**⚠️ 평가자 체크포인트 (필압)**
- 픽셀 루프 추가비용은 곱1·시프트1뿐(스탬프 단위 `scaleFx` 1회 계산, **LUT 재생성 없음**). 60fps 유지 확인(큰 굵기·넓은 칸).
- coalesced 각 샘플마다 `pressure(of:)` 호출 → 폭주 시 비용. force 미지원/`maxForce==0` 폴백(0 나눗셈 가드) 동작 확인.
- 손가락·마커·지우개·penOnly·줌팬 경로 동작 동일(필압은 펜+색연필일 때만 농담 변화).

### 캔버스 줌/팬 (Pinch-to-Zoom & Pan) — P1 추가 (2026-06-04)

> 배경: 세밀한 선 안에 색칠하려면 도안을 가까이 당겨 볼 필요가 있다. 두 손가락 핀치로 확대하고, 확대 상태에서 이동하며 칠하다가, 더블탭으로 원래 크기로 돌아온다.

**확정 (이번 세션)**
1. **핀치 줌** — 두 손가락 오므리기/벌리기로 **1x(기본) ~ 최대 3x** 범위 확대/축소. 1x 미만 축소 불가.
2. **패닝** — 줌인 상태에서 두 손가락 드래그로 캔버스 이동. 줌과 세트.
3. **더블탭 리셋** — 두 손가락 더블탭 → 스프링 애니메이션으로 **1x + 원위치 즉시 복귀**.
4. **툴 레일 고정** — 우측 세로 툴 레일·팔레트 패널은 줌/팬에 영향받지 않고 고정.
5. **색칠 그대로** — 확대 상태에서도 한 손가락/펜 드래그는 색칠(기존과 동일). 두 손가락만 줌/팬 담당.

**제스처 상세**

| 제스처 | 동작 |
|---|---|
| 두 손가락 벌리기/오므리기 | 두 손가락 중점(midpoint) 기준 확대/축소 |
| 두 손가락 드래그 | 캔버스 패닝(이동) |
| 두 손가락 더블탭 | 1x + 원위치 리셋 (스프링 애니메이션) |
| 한 손가락/펜 드래그 | 색칠 (기존 그대로) |

**배율 및 경계 처리**
- 배율 범위: 1.0x ~ 3.0x. 경계에서 바운스 없이 클램핑.
- 줌 중심: 두 손가락 중점 기준. (화면 중앙이 아닌 손가락이 찍힌 위치 기준으로 확대)
- 패닝 경계: 확대된 캔버스가 뷰 바깥으로 완전히 벗어나지 않도록 오프셋 클램핑. (빈 배경이 과도하게 드러나지 않게)

**UI 표시**
- 1x 초과 상태일 때만 캔버스 좌상단 코너에 작은 배율 뱃지 표시 (예: **"2×"**). 1x일 땐 숨김.
- 뒤로가기 버튼과 위치가 겹치지 않게 배치(위치·크기는 디자인에서 확정).

**"펜으로만 색칠하기" 설정과의 관계**
- penOnly가 ON이어도 **두 손가락 줌/패닝은 허용**. 손가락으로 색칠만 막고, 탐색 제스처(핀치·패닝)는 항상 손가락으로 동작.

**구현 메모 (개발 인계)**
- `scaleState: CGFloat`(1.0~3.0), `offsetState: CGSize`를 캔버스 뷰에 추가.
- 캔버스 콘텐츠 영역에만 `.scaleEffect(scaleState).offset(offsetState)` 적용. 툴 레일은 이 변환 **밖**.
- 제스처 분기: 두 손가락 = 줌/팬, 한 손가락/펜 = 색칠. `UIGestureRecognizerDelegate.numberOfTouches` 또는 SwiftUI `SimultaneousGesture` + 터치 수 조건으로 분리.
- 오프셋 클램핑: `maxOffset = canvasSize * (scaleState - 1) / 2`.
- 더블탭: SwiftUI `TapGesture(count: 2)`(두 손가락) 또는 `UITapGestureRecognizer(numberOfTapsRequired:2, numberOfTouchesRequired:2)` → `withAnimation(.spring()) { scaleState = 1.0; offsetState = .zero }`.
- 색칠 스트로크 좌표: 줌인 상태에서 드래그 위치를 **캔버스 로컬 좌표로 역변환**해 RegionPaintEngine에 전달해야 함. (`(touchPoint - offset) / scale`)
- 줌인 상태는 화면 이탈/재진입 시 **항상 1x로 초기화** (저장하지 않음).

**⚠️ 평가자 체크포인트 (줌/팬)**
- 핀치 도중 색칠 제스처가 잘못 발동되지 않는지 (제스처 충돌 가드).
- 확대 상태에서 스트로크가 **올바른 캔버스 좌표로 변환**되는지 — 줌/오프셋 보정 없으면 색칠 위치가 어긋남.
- 배율 경계(1x / 3x)에서 클램핑이 부드럽게 동작하는지.
- 줌인 중 팔레트 슬라이드 패널 오픈·닫힘, 색 선택이 정상 동작하는지.
- penOnly ON 상태에서 두 손가락 핀치/패닝이 여전히 작동하는지.

**범위 밖 (이번에 안 함)**
- 1x 미만 축소(도안 축소 보기)
- 한 손가락 더블탭 리셋 (색칠 제스처와 충돌 가능성)

- 줌인 상태 저장 (재진입 시 항상 1x 리셋)

**확정 (이번 세션)**
1. **위치·형태** — 우측 세로 툴 레일 **바로 아래**, 살짝 간격을 두고 **원형 휴지통 버튼**. 지름 = **레일 가로 폭과 동일**(현재 96pt). SF Symbol `trash`(또는 `trash.fill`) 사용.
2. **톤** — **레일과 같은 크림/카드 톤**(`Theme.card` 배경 + `cardBorder` 보더 + `softShadow`), 아이콘은 차분한 회색(`Theme.ink`/`subText` 계열). 위험 강조색(빨강/코랄)은 쓰지 않음 — 평소엔 눈에 띄지 않게.
3. **확인 다이얼로그** — 탭하면 **바로 지우지 않고** 확인 다이얼로그를 먼저 띄운다. 문구 예: "**○○의 색칠을 모두 지울까요?**"(○○ = **도안 이름이 있으면 그 이름, 없으면 "이 그림"** 으로 대체), 버튼 **[취소] / [초기화]**. **[초기화]는 파괴적(destructive) 스타일**(빨강 라벨)로 표시. 확인 시에만 실제 초기화. (삭제 확인 다이얼로그와 동일한 안전장치 톤.)
4. **초기화 동작 (확인 시)**
   - 캔버스 **색칠 버퍼를 비운다**(`RegionPaintEngine`의 채색 버퍼 clear) → 화면엔 빈 도안(라인아트)만 남는다.
   - 저장된 **작업물(Artwork)의 진행 데이터를 삭제**한다. 구현은 **해당 Artwork 행 삭제**(현재 프로필×도안의 작업물 제거)로 통일 → 갤러리에서 이 도안은 **빈 도안(원본) 썸네일**로 표시된다. (초기화 직후 아무것도 안 칠하고 나가도 갤러리가 '진행 중'으로 남지 않음.)
   - 초기화 후 **사용자는 캔버스에 그대로 머문다**(화면 이동 없음). 이어서 새로 칠하면 그때 작업물이 다시 생성·저장된다(기존 `persist` 경로 재사용).
5. **유지되는 것** — 현재 **선택 색·도구(마커/색연필)·굵기·지우개 상태는 그대로**. 초기화는 *칠해진 결과*만 비운다(도구 설정은 안 건드림).

**구현 메모 (개발 인계)**
- 초기화는 캔버스 엔진에 "비우기" 명령을 전달해야 함 → 현재 `CanvasArea`/`DrawingCanvas`가 엔진을 들고 있으므로, **reset 토큰(또는 트리거 바인딩)** 을 내려보내 엔진 버퍼를 clear하고 한 번 다시 렌더하는 방식 권장. (뷰 식별자 교체로 강제 재생성하는 방법도 가능하나, 도구 상태 보존·라인아트 재로딩 비용 고려해 토큰 방식이 가벼움.)
- Artwork 삭제는 `context.delete(art)` 후 `try? context.save()`. 삭제 후 `existing`은 nil이 되고, 자동저장 디바운스가 **삭제 직후 빈 버퍼를 다시 저장해 되살리지 않도록** 주의(초기화 시 진행 중인 저장 작업 취소/무시).
- **되돌리기 없음**(이번 범위): 확인 다이얼로그가 유일한 안전장치. (전역 undo는 화면 3 '되돌리기' P1 후보와 함께 추후.)

**⚠️ 평가자 체크포인트 (초기화)**
- 버퍼 clear는 **`memset`/전체 재할당 1회**로 끝내고, 매 프레임 도는 경로에 들어가지 않게.
- 삭제 후 **자동저장이 빈 작업물을 되살리지 않는지**(레이스) 확인 — 디바운스 타이머/`saver.flush()`와 충돌 주의.
- 다이얼로그 띄운 채 뒤로가기/연타 시 **중복 삭제·dangling 상태** 가드.

### 사용자 흐름
```
도안 탭 → 캔버스 진입(이전 진행 복원) → 색칠 → 레일 저장 버튼 탭 → 즉시 저장 + "저장됨" 토스트

```

## 앱 설정 화면 (App Settings) — P1 *(추가 2026-06-04)*

### 목적
기기 전체에 적용되는 앱 동작을 조정한다. 부모가 한 번 설정하면 모든 프로필에 동일하게 적용된다.

### 진입 방법
- **화면 1(프로필 선택)** 한켠 코너에 **기어(⚙) 버튼** 배치.
- 탭 → **모달 시트(sheet)** 로 설정 화면 등장. (별도 풀스크린 화면 아님 — 시트로 충분.)

### 설정 항목

#### 1. 펜으로만 색칠하기 — P1 *(신규, 2026-06-04)*

| 항목 | 내용 |
|---|---|
| 토글 라벨 | "펜으로만 색칠하기" |
| 부제(선택) | "켜면 Apple Pencil로만 색칠할 수 있어요" |
| 기본값 | **ON** (앱 첫 설치 시 켜진 상태) |
| 저장 위치 | `UserDefaults` (기기 전체 공용 — 프로필별 아님) |
| 적용 범위 | 화면 3(색칠 캔버스) |

**동작 상세**
- **ON**: 색칠 캔버스에서 **Apple Pencil 입력만 색칠에 반응**. 손가락 터치는 색칠 무시 (스크롤·줌·UI 버튼은 그대로 동작).
- **OFF**: 손가락으로도 색칠 가능.

**구현 메모 (개발 인계)**
- `AppSettings`(또는 `UserDefaults` 직접 키) `penOnly: Bool` 플래그.
- 색칠 캔버스의 `DragGesture` / Pencil 인식 로직에서 `pencilOnly == true && input.type == .touch` 이면 무시.
- `RegionPaintEngine`이 커스텀 래스터 엔진이므로, **제스처 진입 시점**(예: `onChanged`, `onEnded`)에서 `inputType`(pencil / touch)을 구분해 `pencilOnly` 일 때 touch 경로를 얼리 리턴.
- `UITouch.type == .pencil` or `.stylus` → Pencil 입력으로 판단.

**⚠️ 평가자 체크포인트**
- 손가락 터치를 무시할 때 기존 UI 버튼(레일·팔레트·뒤로 등) 터치 반응에 영향이 없는지.
- ON/OFF 전환 시 진행 중 스트로크가 갑자기 끊기거나 엔진 상태가 오염되지 않는지.

### 데이터 / 저장
- `UserDefaults.standard.bool(forKey: "penOnly")` (기본값 `true`).
- 앱 종료·재시작 후에도 유지.
- 프로필 삭제 시 영향 없음(기기 전역 설정이므로).

### 레이아웃 지침 (디자인으로)
- 단일 시트 안에 토글 하나(첫 설정). 향후 설정이 늘어나도 같은 시트 안에 추가.
- 디자인 톤은 앱 전체와 동일(크림/파스텔).
- 설정 버튼(기어)은 화면 1의 '사용자 추가' 버튼과 **반대쪽 코너**에 배치. 위치는 디자인에서 확정.

### 범위 밖 (이번에 안 함)
- 비밀번호/잠금 설정 (부모 보호장치) — 별도 기획.
- 언어·테마·폰트 크기 등 기타 설정 — 추후.

## 기능: iPhone 부모 제어판 — Parental Remote Timer (P1)

### 목적
부모가 자신의 iPhone으로 아이가 색칠 중인 iPad 앱을 원격 제어한다.
v1은 **타이머 하나만**: 설정한 시간이 다 되면 iPad 앱이 즉시 홈(프로필 선택) 화면으로 복귀한다.

### 앱 구조 (유니버설 바이너리)
- **동일 앱 하나**를 iPhone과 iPad에 모두 설치 (별도 앱 불필요).
- 실행 시 기기 종류를 자동 감지:
  - **iPad** → 기존 색칠 앱 그대로
  - **iPhone** → 부모 제어판 화면만 표시
- `UIDevice.current.userInterfaceIdiom`으로 분기, `#if` 컴파일 분기 불필요.

### 연결 방식
- **Apple MultipeerConnectivity 프레임워크** — WiFi Direct·블루투스 자동 선택.
- 인터넷 불필요. 같은 공간이면 동작.
- **1:1 구조**: 부모 iPhone 1대 ↔ 아이 iPad 1대. 여러 대 동시 제어는 이번 범위 밖.
- 최초 앱 실행 시 한 번 페어링 → 이후 자동 재연결.

### 사용자 흐름
```
[부모 iPhone]
앱 실행 → 페어링(iPad 탐색·선택·연결) → 제어판
  └─ 타이머 분 단위 설정 → [시작]
       ├─ 남은 시간 표시 + [취소] 가능
       └─ 만료 → iPad에 '즉시 복귀' 명령 전송

[아이 iPad - 색칠 캔버스]
연결 상태에서 타이머 진행 중:
  ├─ 남은 시간이 항상 화면 한쪽에 표시
  ├─ 5분 남으면 → 1차 알림 (시각적 강조·색 변화 등, 구체 방법은 디자인에서 결정)
  ├─ 1분 남으면 → 2차 알림 (더 강한 강조)
  └─ 0초 → 즉시 프로필 선택 화면으로 이동
```

### 기능 상세

#### 연결/페어링 — P0
- iPhone 앱 첫 실행 → 근처 iPad 자동 탐색 → 목록에서 선택 → 연결.
- 연결 상태를 제어판 상단에 항상 표시 (연결됨 / 연결 중 / 연결 끊김).
- 연결이 끊겼다가 복구되면 자동 재연결 시도.

#### 타이머 설정 — P0
- **프리셋**: 10분 / 15분 / 20분 / 30분 버튼 (빠른 선택).
- **직접 입력**: 분 단위 숫자 입력 가능 (프리셋에 없는 시간 대응).
- [시작] 탭 → 카운트다운 시작. iPad에 타이머 시작 명령·총 시간 전송.
- 진행 중 [취소] → 타이머 해제. iPad에 취소 명령 전송 → iPad 타이머 표시 사라짐, 색칠 계속.

#### 타이머 설정 UX — 선택 상태 표시 (확정 2026-06-04)

**상태 표시 규칙 (텍스트 필드가 단일 진실 공급원)**
1. **프리셋 버튼 탭** → 텍스트 필드 값을 해당 분 수로 **자동 덮어씀** + 그 버튼에 **강조 상태(선택됨)** 표시. 이전에 입력한 직접 입력 값은 버려진다.
2. **텍스트 필드 직접 입력** → 입력 값이 프리셋 중 하나와 일치하면 **그 버튼이 강조됨**. 일치하지 않으면 모든 프리셋 버튼이 비강조 상태.
3. **텍스트 필드를 비우거나 0 입력** → 모든 프리셋 버튼 비강조. [시작] 버튼 비활성.
4. **[시작] 탭** → 항상 **텍스트 필드 현재 값**을 기준으로 타이머 시작 (프리셋 눌렀더라도 그 후 텍스트 필드를 수정했으면 수정값 우선).

**프리셋 버튼 강조 스타일**
- 비선택: 테두리만 있는 빈 버튼 (크림/카드 톤).
- 선택됨: **코랄/강조색 배경** 채움 + 텍스트 색 반전 (흰색). 전환은 부드러운 애니메이션(`withAnimation(.easeInOut)`).

**구현 메모 (개발 인계)**
- `selectedMinutes: Int?` 상태 하나로 관리. 프리셋 탭 시 `selectedMinutes = preset` + `inputText = "\(preset)"`. 텍스트 필드 변경 시 `selectedMinutes = Int(inputText)` (파싱 실패 시 nil).
- 프리셋 버튼마다 `preset == selectedMinutes` 비교해 배경색·텍스트색 전환.

#### iPad 타이머 표시 — P0
- 색칠 캔버스 화면에서 **남은 시간을 항상 표시** (위치·크기는 디자인에서 결정).
- **표시 형식 = "X분 XX초"** (아이 친화적) — 숫자만 나열하는 MM:SS 포맷(04:32 등) 대신 **"4분 32초"** 처럼 단위를 붙여 읽기 쉽게. 1분 미만은 "59초" 처럼 초만 표시. 디자인적으로도 친근한 폰트·크기 사용 (§22 TimerChip 참조).
- 타이머가 없거나 취소되면 표시 숨김.
- **5분 남으면 1차 알림**: 시각 방법은 디자인 재량 (색상 강조, 펄스 애니메이션, 배너 등).
- **1분 남으면 2차 알림**: 1차보다 더 눈에 띄는 방법 (색상 더 강렬하게, 흔들림 등).
- **0초 → 즉시 프로필 선택 화면(화면 1)으로 이동** — `NavigationStack path`를 `removeAll()`로 초기화해 갤러리(화면 2)를 거치지 않고 곧바로 사용자 선택 화면으로 복귀. 색칠 저장(`saver.flush()`) 후 이동.

#### 연결 끊김 중 타이머 처리 — P0
- **iPhone**: 연결 끊겨도 타이머는 계속 카운트.
- **iPad**: 연결 끊겨도 수신한 만료 예정 시각(절대시각)을 기준으로 **혼자 카운트다운 유지**. 재연결 여부와 무관하게 만료 시 복귀.
- 구현: 타이머 시작 시 `Date` 기반 만료 시각을 iPad에 전달 → iPad는 로컬 타이머로 독립 관리.

### 데이터 / 상태
- 타이머 상태(`timerEnd: Date?`)는 앱 내 메모리만 — 앱 종료 시 초기화. (종료 후 재진입은 타이머 없는 상태로 시작.)
- 페어링 기기 정보(이름·Peer ID)는 UserDefaults에 보관해 재실행 시 자동 재연결 시도.

### 플랫폼 고려사항
- iPhone UI: 세로 전용(Portrait), 단순 제어판이므로 좁은 화면 최적화.
- iPad UI: 기존 색칠 앱과 동일. 타이머 표시 영역만 추가.
- MultipeerConnectivity는 백그라운드 진입 시 연결이 끊길 수 있음 → 앱이 포그라운드일 때만 동작을 보장하는 것으로 v1 범위 설정.

### 비기능 요구사항
- 타이머 오차: ±2초 이내 (로컬 타이머 기반이라 네트워크 지연 무관).
- 연결 수립 시간: 탐색~연결 완료 10초 이내 (같은 WiFi 환경 기준).

### 범위 밖 (v1 제외)
- 즉시 강제 종료(타이머 없이 바로 홈으로) — v2 후보.
- 팔레트·도구 잠금/제한 — v2 후보.
- 실시간 색칠 화면 미리보기(라이브 스트리밍) — 난이도 높음, 추후 검토.
- 여러 iPhone이 동시에 제어 — v2 후보.
- 백그라운드 타이머 유지(iPhone 앱을 내리고도 iPad가 작동) — v2 후보.

### 확정 사항
1. **알림 방법**: 시각 전용 — 소리 없음. 5분·1분 알림의 구체 시각 표현은 디자인에서 결정. *(2026-06-04)*
2. **만료 시 저장**: 타이머 만료 → **색칠 자동 저장 후** 즉시 홈으로 이동. 기존 `saver.flush()` 경로 재사용. *(2026-06-04)*
3. **iPhone 제어판 표시**: 아이의 현재 프로필·도안명 **표시 안 함**. 타이머 제어만 노출. *(2026-06-04)*
4. **iPad 타이머 표시 형식**: `"X분 XX초"` (아이 친화적) — 분+초 단위 텍스트. 1분 미만은 초만. MM:SS 콜론 포맷 폐지. *(2026-06-04)*
5. **만료 시 이동 목적지**: 갤러리(화면 2)가 아닌 **프로필 선택 화면(화면 1)**. `NavigationStack path.removeAll()`로 스택 전체 초기화. *(2026-06-04)*
6. **타이머 설정 선택 상태 표시**: 프리셋 탭 → 텍스트 필드 자동 반영 + 해당 버튼 강조(코랄 배경). 텍스트 직접 입력 값이 프리셋과 일치하면 자동 강조. 불일치하면 모든 버튼 비강조. 텍스트 필드가 항상 최종 값. *(2026-06-04)*
