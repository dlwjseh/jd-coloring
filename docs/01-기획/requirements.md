상태: 진행 중 — 화면 1(사용자 선택) 확정, 이후 화면 미정

## 제품 개요 (작성 중)
- **한 줄 요약**: (추후 보강) 온 가족/여러 사용자가 각자 프로필로 즐기는 컬러링 앱
- **타깃 사용자**: 다중 사용자 환경(가족·아이 포함 추정) — 프로필 기반 구조

## 화면 1: 사용자 선택 (User Selection) — P0

### 목적
앱 실행 시 가장 먼저 나타나는 화면. 사용할 프로필을 고르거나 새로 만든다. (넷플릭스·iPad 프로필 선택과 유사)

### 레이아웃
- 등록된 사용자가 **원형 프로필**로 **가로 한 줄** 배치
- **정렬**: 기본은 **가운데 정렬**(프로필이 화면 폭 안에 다 들어오면 가로 중앙으로 모음). **화면을 넘치면 좌측 정렬 + 좌우 드래그(수평 스크롤)** 로 전환
- 각 프로필: 원형 이미지 + 그 아래 **이름** 라벨
- 화면 **구석에 '사용자 추가' 버튼** 배치 (위치는 디자인에서 확정 — 우상단 모서리 또는 프로필 줄 끝의 '+' 원 제안)

### 사용자 흐름 (User Flow)
```
앱 실행
  ├─ 등록된 사용자 있음 → 프로필 가로 목록 표시
  │     ├─ 프로필 탭        → 다음 화면 진입  ※ 진입 후 동작은 다음 단계에서 기획
  │     └─ 프로필 롱프레스  → [수정] / [삭제] 메뉴
  │            ├─ 수정 → 사용자 추가 화면 재사용(이름·이미지 변경)
  │            └─ 삭제 → 확인 다이얼로그 → 확인 시 삭제
  └─ 등록된 사용자 없음(첫 실행) → 곧바로 '사용자 추가' 화면
```

### 기능 상세

**1) 사용자 추가 — P0**
- 진입: 추가 버튼 탭, 또는 첫 실행(사용자 0명) 시 자동
- 입력: **이름**(텍스트) + **프로필 이미지**
- 이미지 소스: **사진 앨범/갤러리에서 선택** (카메라·기본 아바타는 이번 범위 밖)
  - iOS/iPadOS 사진 라이브러리 접근 권한 처리 필요
  - 원형 표시에 맞춰 **정사각/원형 크롭** 단계 필요
- 저장 시 프로필 목록에 새 원형으로 추가

**2) 사용자 수정 — P0**
- 진입: 프로필 롱프레스 → [수정]
- 추가 화면을 재사용해 이름·이미지를 변경 후 저장

**3) 사용자 삭제 — P0**
- 진입: 프로필 롱프레스 → [삭제]
- **확인 다이얼로그** 표시("정말 삭제할까요?") → 확인 시에만 삭제
- (해당 사용자의 작업물 처리 정책은 작업물 기능 기획 시 결정 — 열린 질문)

**4) 프로필 선택(진입) — 다음 단계**
- 프로필 탭 시 다음 화면으로 이동. 이동 후 화면/동작은 **별도 기획 예정**

### 화면 전환 / 애니메이션 — P1

전부 UI 레벨 연출이므로 **순수 SwiftUI로 구현**(Metal 불필요). 느낌은 빠르고 경쾌하게("샤악/스윽"), 끝에 살짝 스프링으로 정착.

**A1. 화면 진입 — 프로필 등장**
- **트리거**: 사용자 선택 화면에 올 때마다 (앱 실행 + 다른 화면에서 복귀 시 매번 재생)
- **동작**: 프로필 원들이 화면 **오른쪽 바깥 → 제자리로 슬라이드 인** (오른쪽에서 왼쪽 방향), 페이드 인 동반
- **방식**: **순차(stagger)** — 각 프로필이 약간의 시차(제안: 50~80ms 간격)를 두고 하나씩 "주르르르" 등장
- 끝에 가벼운 스프링 정착감

**A2. 등록·수정 진입 — 프로필 흩어짐 + 폼 등장**
- **트리거**: '추가' 버튼 탭, 또는 롱프레스 → [수정]
- **1단계**: 현재 프로필들이 **좌·우로 갈라져 화면 밖으로 흩어짐**(가운데를 비움)
- **2단계**: 비워진 가운데로 **수정 UI(이름 입력칸 + 이미지 선택)가 스윽 등장** — 위/아래 방향은 디자이너 결정(제안: 아래에서 위로)
- 두 단계는 살짝 겹쳐(overlap) 자연스럽게 이어짐
- **역방향(reverse)**: 저장/취소로 빠져나갈 때 폼이 사라지고 프로필이 **다시 가운데로 모이며** 복귀

**구현 메모(개발자용)**
- SwiftUI `transition` + `withAnimation`(spring), stagger는 항목 인덱스 기반 delay로 구현
- `matchedGeometryEffect` / `phaseAnimator` 활용 검토
- ⚠️ **평가자 체크포인트**: 타이머 남발로 stagger 구현 금지, 프로필 多일 때 동시 애니메이션 부하, "매 진입 재생"이라 무거우면 바로 체감됨 → 가볍게 유지

### 데이터/저장 요구사항
- 프로필 = { id, 이름, 이미지 } 를 **기기 로컬에 영구 저장**
- 선택한 갤러리 이미지를 앱 내부 저장소에 복사·보관(원본 삭제돼도 유지)
- 프로필 표시 순서(추가 순/최근 사용 등)는 추후 결정 — 열린 질문

### 플랫폼 고려사항
- **iPad**: 터치로 탭·롱프레스·수평 드래그. 사진 앨범 접근.
- **Mac**: 마우스/트랙패드 — 롱프레스 대응 방식 필요(우클릭 컨텍스트 메뉴 등으로 대체 검토), 수평 스크롤은 트랙패드 제스처/스크롤. 사진 선택은 파일/사진 앱 연동.

### 비기능 요구사항
- 수평 스크롤·드래그가 부드럽게(60fps) 동작
- 프로필 이미지 다수여도 썸네일 다운샘플링으로 메모리 절약

### 범위 밖 (Out of Scope) — 이번 화면에서 안 함
- 카메라 촬영 / 기본 제공 아바타
- 어른 잠금·비밀번호 등 강한 보호장치(삭제 확인 다이얼로그까지만)
- 프로필 선택 이후의 화면들

### 열린 질문 (Open Questions)
1. 프로필 **최대 개수** 제한이 있는가? (기본: 제한 없음, 스크롤로 처리)
2. 프로필 **정렬 순서** 기준은? (추가순 / 최근 사용순)
3. 사용자 **삭제 시 그 사용자의 작업물**도 함께 삭제하는가?
4. '사용자 추가' 버튼의 정확한 위치/형태 (→ 디자인에서 확정)

## 화면 2: 색칠 도안 갤러리 (Coloring Gallery) — P0

### 목적
프로필 선택 후 진입하는 화면. **색칠할 도안들이 그리드로 펼쳐지고**, 칸을 탭하면 색칠을 시작한다. 새 도안을 **구석의 추가 버튼**으로 업로드한다. (사진첩 앱의 격자 + 컬러링 보관함 느낌)

### 핵심 결정 (이번 세션 확정)
1. **도안은 공유 · 작업물은 개인** — 누군가 올린 도안은 온 가족(모든 프로필)이 함께 보고 각자 색칠. 색칠 결과(작업물)는 프로필별로 따로 저장.
2. **그리드 칸 = 현재 프로필의 진행 상태** — 내가 칠하던 도안은 칠한 모습, 안 건드린 도안은 원본(빈) 도안 썸네일.
3. **업로드 = 사진 그대로 도안 등록** — 사진을 골라 미리보기로 확인한 뒤 가공 없이 도안으로 추가. (라인아트 자동 변환은 결과 품질이 들쭉날쭉해 **제거** — 2026-06-01 결정)

### 데이터 모델 (개발자 인계)
- **도안(Template)** — 전역 공유: `{ id, 도안 이미지(영구저장), 추가일, 추가한 프로필 id(옵션) }` (변환 제거로 '원본 이미지' 별도 보관 필드 불필요)
- **작업물(Artwork)** — (프로필 × 도안)당 최대 1개: `{ id, 프로필 id, 도안 id, 색칠 진행 데이터, 진행 썸네일, 마지막 수정일 }`
  - 색칠 "진행 데이터"의 구체 형식은 **화면 3에서 `PKDrawing` data로 확정**(`Artwork.progressData`). 이번(갤러리) 화면은 *썸네일만* 표시.
- 그리드 셀 표시 규칙: 해당 도안에 현재 프로필의 작업물이 있으면 그 진행 썸네일, 없으면 빈 도안 썸네일.

### 레이아웃
- **상단 바**: 현재 프로필(작은 원 + 이름) + **뒤로(프로필 선택으로)**. "○○의 색칠 도안" 정도의 타이틀.
- **본문**: 적응형 그리드(`LazyVGrid`, 화면 폭에 따라 열 수 자동). 각 셀은 **라운드 카드**(도안 썸네일 + 부드러운 그림자), 화면 1과 같은 크림/파스텔 톤.
- **구석 추가 버튼**: 화면 1의 추가 버튼과 동일 톤·위치(우하단 코너 플로팅 원, "도안 추가").
- **빈 상태(도안 0개)**: 안내 문구 + 추가 유도(첫 진입 시 곧바로 업로드로 유도할지 디자인에서 결정).

### 사용자 흐름
```
프로필 선택 → 갤러리 진입
  ├─ 도안 있음 → 그리드 표시
  │     ├─ 셀 탭         → 색칠 캔버스 진입  ※ 캔버스는 다음 단계 기획
  │     ├─ 셀 롱프레스   → [내 색칠 초기화] / [도안 삭제] (※ 공유 삭제 정책 주의 — 열린 질문)
  │     └─ 추가 버튼 탭  → 사진 선택 → 라인아트 변환 미리보기 → 저장
  └─ 도안 없음 → 빈 상태(추가 유도)
  └─ 뒤로 → 화면 1(프로필 선택)
```

### 도안 업로드 — P0
- 사진 선택: `PhotosPicker`(화면 1과 동일 경로).
- 선택하면 **'도안 만들기' 시트에 크게 미리보기** → [취소] / [도안 저장]으로 확인 후 추가. (사진 1장 = 도안 1개)
- **라인아트 자동 변환 제거(2026-06-01)**: 사진을 가공 없이 그대로 도안으로 등록한다. 사용자가 이미 선이 있는 색칠 도안 이미지를 올린다고 가정.
- ⚠️ **평가자 체크포인트**: 업로드 이미지는 다운샘플해 보관. **그리드 표시용 썸네일**과 (색칠용) 표시 이미지의 해상도를 분리(검수 increment4 지적). 디코딩·IO는 백그라운드.

### 화면 전환 / 애니메이션 — P1

**G1. 프로필 ↔ 갤러리 전환 — 흩어짐 + 밑에서 올라옴** *(확정 2026-06-02, 디자인 스펙 §12 / 목업 `16-anim-G1-slide.svg`)*
- **트리거**: 프로필 선택 화면에서 프로필 원을 **탭** → 갤러리 진입. '뒤로' → 복귀.
- **정방향 동작**: ① 프로필 화면 요소가 **빠짐**(타이틀 위로, 프로필 원들 좌·우로 흩어짐, 추가 버튼 아래로) → ② 갤러리 **도안 카드가 밑에서 우수수 올라옴**(stagger) + 타이틀·버튼 등장. 화면 전환 자체는 **무애니**(가로 슬라이드 제거), 두 화면이 같은 배경을 공유해 매끄럽게 이어짐.
- **역방향(reverse)**: 정방향의 **대칭 역재생** — 갤러리 빠짐(타이틀 위로, 카드·버튼 아래로) → 프로필 복귀(흩어졌던 것이 제자리로).
- **배포 타깃**: **iOS 18 / macOS 15로 상향**(확정).
  - ※ 초기엔 네이티브 zoom 전환(`.navigationTransition(.zoom)`)으로 잡았으나 펼침 느낌이 어색해 위 방식으로 변경(2026-06-02). 타깃 상향은 유지.
- ⚠️ **평가자 체크포인트(반영 완료)**: stagger는 타이머 금지(`.animation().delay()`), 전환 지연은 취소 가능한 `Task.sleep`, 연타/재진입 가드(중복 push·dangling dismiss 차단), **Reduce Motion 대응**(이동·stagger 생략), 도안 多 시 60fps 유지.

### 범위 밖 (이번 화면에서 안 함)
- 실제 색칠 캔버스(붓·채우기·되돌리기 등) — 별도 화면 기획.
- 도안 카테고리/검색/정렬 옵션 UI(초기엔 단순 최신순).
- ~~라인아트 변환~~ 기능 **제거**(2026-06-01). 추후 필요 시 별도 기획.

### 열린 질문
1. **공유 도안 삭제 권한** — 아무 프로필이나 삭제 가능? 올린 사람만? 삭제 시 다른 사람 작업물도 사라지므로 확인 다이얼로그 강화 필요.
2. **정렬 기준** — 최근 추가순 / 내가 최근 색칠한 순.
3. **셀 롱프레스 메뉴** — "내 색칠 초기화"(작업물만 삭제, 도안 유지) 포함 여부. → **의미 확정(2026-06-02)**: '내 색칠 초기화' = 해당 작업물(Artwork) 삭제 → 빈 도안 썸네일로 복귀. 캔버스 안 '색칠 초기화'(화면 3)와 **동일 동작**. (갤러리 메뉴 노출 여부 자체는 추후.)
4. 업로드 이미지 **크롭/회전 보정** 제공 여부(이번 범위 밖, 추후 검토).

## 화면 3: 색칠 캔버스 (Coloring Canvas) — P0

### 목적
그리드에서 도안을 탭하면 열리는 **실제 색칠 화면**. 선택한 도안 위에 브러시로 색칠하고, 진행 상태가 자동 저장돼 갤러리에 '진행 중'으로 반영된다.

### 핵심 결정 (이번 세션 확정)
1. **색칠 방식 = 브러시 직접 칠하기** — 손가락/펜으로 자유롭게 칠. (탭-채우기/flood fill은 이번 범위 밖, 추후)
2. **입력 = Apple Pencil 우선(필압·기울기)** → **PencilKit(`PKCanvasView`)** 기반. iPad 펜 경험 1순위, Mac은 포인터로 대체.

- **상시(우측 레일)** — *개정 2026-06-01*: 현재 색 + **'색' 버튼**(전체 펼치기). ~~최근색 5~6개~~ **레일에서 제거**(세로 폭 절약). 최근색은 펼침 패널 상단에서 보여준다(아래).
- **펼침 패널(우측 슬라이드)** — *개정 2026-06-01*: '색' 버튼 탭 → 화면 우측에서 **슬라이드 패널**이 등장. 구성: 상단에 **최근 사용색 줄**(5~6개) + 그 아래 72색을 **8행 × 9열** 그리드(차트 배치 순서 계승: 노랑→주황·빨강→핑크·보라→파랑→청록·초록→초록·갈색→갈색·무채색→무채색). 색 선택 시 닫히고 최근색에 반영. 패널은 캔버스 위 **오버레이**라 도안 크기에 영향 없음.

- 선택 색은 크게 + 진한 링으로 표시. 흰색·크림 등 옅은 색은 보더로 구분.

### 채색 도구 — 마커 + 색연필 — P1 추가 (2026-06-02)

> 배경: 현재 채색은 영역(칸)을 잠그고 그 칸 픽셀을 **완전 불투명(alpha 255) 단색**으로 채운다(`RegionPaintEngine.stamp`). 납작한 면이라 결·농담이 없다. "좀 더 색연필 질감"을 원한다는 요청 → **색연필 도구를 별도로 추가**하고 기존 단색 채움은 '마커'로 유지해 둘을 전환한다.

**확정 (이번 세션)**
1. **도구는 두 가지, 전환식** — 기존 단색 채움 = **마커**(또는 '채우기' 톤, 명칭은 디자인 재량), 신규 = **색연필**. 우측 레일의 도구 전환 컨트롤로 선택. 둘 다 **영역 가두기(칸 안에서만 칠해짐)·굵기 3단계·지우개**는 동일하게 적용.
2. **색연필 핵심 동작 = 쌓임(build-up) + 종이결 입자.**
   - **쌓임(반투명 누적)**: 한 번 칠하면 옅게, 같은 칸을 **여러 획으로 덧칠하면 점점 진해진다**(최대 = 그 색의 진한 값에서 멈춤). 가장 색연필다운 특성.
   - **종이결/거친 입자**: 칠한 면에 미세한 종이 질감 입자가 비쳐, 균일한 면이 아니라 까끌한 결이 보인다.
3. **세기 = 은은하게** — 멀리서 보면 거의 채워 보이고, 가까이서 보면 살짝 결·농담이 느껴지는 수준. 과한 노이즈/얼룩짐 금지.

**색연필 동작 상세 (개발 인계)**
- **반투명 스탬프**: 마커처럼 alpha를 255로 덮어쓰지 말고, 색을 **부분 불투명도로 위에 올려(source-over 알파 합성)** 칠할수록 누적되게 한다. (옅은 1패스 → 덧칠로 진해짐.)
- **한 획 안에서는 균일, 획을 새로 그으면 진해짐**: 하나의 연속 드래그(다운→업) 동안에는 같은 픽셀이 겹쳐도 **이중 적용되지 않게** 한다(획 내 커버리지 마스크로 1회만 반영). 안 그러면 스탬프가 촘촘히 겹치는 곳만 얼룩진다. 새 획을 그어 그 위에 또 칠하면 그때 진해진다.
- **종이결 입자**: 미리 만든(또는 작은 타일) **그레인 노이즈 마스크**로 픽셀별 불투명도를 살짝 변조. '은은하게'이므로 변조 폭은 작게(예: 불투명도의 ±소량). 매 스탬프마다 노이즈를 새로 생성하지 말고 **고정 텍스처를 화면 좌표에 고정해 룩업**(움직여도 결이 떨림 없이 종이에 고정된 느낌).
- **영역 경계는 그대로 보존**: 경계(라벨 0) 픽셀은 색연필도 절대 안 칠함 → 검은 선 유지.
- **지우개**: 색연필 칠도 지워야 하므로 기존 지우개(해당 칸 픽셀 alpha 0)로 동일 처리.
- **저장/썸네일**: 반투명 누적 결과가 색칠 버퍼 픽셀에 그대로 반영되므로 기존 저장 경로(`progressData` PNG + 합성 썸네일) 변경 없음.

**⚠️ 평가자 체크포인트 (색연필)**
- 스탬프 픽셀 루프에 그레인 룩업 + 알파 합성이 추가됨 → **마커보다 픽셀당 비용 증가**. 60fps 유지 확인(특히 큰 굵기·넓은 칸).
- 그레인 텍스처는 **1회 생성 후 재사용**(매 프레임/매 스탬프 재생성 금지).
- "획 내 1회 반영" 커버리지 마스크가 칸 크기만큼 매번 새 버퍼를 alloc하지 않게(재사용/부분 영역만).

**범위 밖 / 추후 (색연필)**
- **필압·기울기 반응**(Apple Pencil): 이번 요청에서 선택 안 됨 → **v1 제외, P1 최우선 후보**. (색연필은 필압과 궁합이 좋아 다음 단계 권장.)
- 크레용·마커펜·수채 등 추가 질감, 결 방향(스트로크 방향성) 표현, 텍스처 종류 선택 — 이번 범위 밖.

### 도구 레일 정비 — 도구 세로화 + 라벨 제거 + 명칭/아이콘 정리 — P1 추가 (2026-06-02)

### 색칠 초기화 (Reset) — P1 추가 (2026-06-02)

> 배경: 지금은 색칠을 되돌릴 방법이 캔버스 안에 없다(되돌리기 v1 미선택). 잘못 칠했거나 처음부터 다시 하고 싶을 때 **현재 도안의 색칠을 한 번에 비우는** 초기화가 필요. 갤러리 롱프레스의 "내 색칠 초기화"(화면 2 열린질문 3)와 **동일한 의미**를, 캔버스 안에서도 바로 실행할 수 있게 한다.

**확정 (이번 세션)**
1. **위치·형태** — 우측 세로 툴 레일 **바로 아래**, 살짝 간격을 두고 **원형 휴지통 버튼**. 지름 = **레일 가로 폭과 동일**(현재 96pt). SF Symbol `trash`(또는 `trash.fill`) 사용.
2. **톤** — **레일과 같은 크림/카드 톤**(`Theme.card` 배경 + `cardBorder` 보더 + `softShadow`), 아이콘은 차분한 회색(`Theme.ink`/`subText` 계열). 위험 강조색(빨강/코랄)은 쓰지 않음 — 평소엔 눈에 띄지 않게.
3. **확인 다이얼로그** — 탭하면 **바로 지우지 않고** 확인 다이얼로그를 먼저 띄운다. 문구 예: "**○○의 색칠을 모두 지울까요?**"(○○ = 도안명 또는 "이 그림"), 버튼 **[취소] / [초기화]**. **[초기화]는 파괴적(destructive) 스타일**(빨강 라벨)로 표시. 확인 시에만 실제 초기화. (삭제 확인 다이얼로그와 동일한 안전장치 톤.)
4. **초기화 동작 (확인 시)**
   - 캔버스 **색칠 버퍼를 비운다**(`RegionPaintEngine`의 채색 버퍼 clear) → 화면엔 빈 도안(라인아트)만 남는다.
   - 저장된 **작업물(Artwork)의 진행 데이터를 삭제**한다. 구현은 **해당 Artwork 행 삭제**(현재 프로필×도안의 작업물 제거)로 통일 → 갤러리에서 이 도안은 **빈 도안(원본) 썸네일**로 표시된다. (초기화 직후 아무것도 안 칠하고 나가도 갤러리가 '진행 중'으로 남지 않음.)
   - 초기화 후 **사용자는 캔버스에 그대로 머문다**(화면 이동 없음). 이어서 새로 칠하면 그때 작업물이 다시 생성·저장된다(기존 `persist` 경로 재사용).
5. **유지되는 것** — 현재 **선택 색·도구(마커/색연필)·굵기·지우개 상태는 그대로**. 초기화는 *칠해진 결과*만 비운다(도구 설정은 안 건드림).

**구현 메모 (개발 인계)**
- 초기화는 캔버스 엔진에 "비우기" 명령을 전달해야 함 → 현재 `CanvasArea`/`DrawingCanvas`가 엔진을 들고 있으므로, **reset 토큰(또는 트리거 바인딩)** 을 내려보내 엔진 버퍼를 clear하고 한 번 다시 렌더하는 방식 권장. (뷰 식별자 교체로 강제 재생성하는 방법도 가능하나, 도구 상태 보존·라인아트 재로딩 비용 고려해 토큰 방식이 가벼움.)
- Artwork 삭제는 `context.delete(art)` 후 `try? context.save()`. 삭제 후 `existing`은 nil이 되고, 자동저장 디바운스가 **삭제 직후 빈 버퍼를 다시 저장해 되살리지 않도록** 주의(초기화 시 진행 중인 저장 작업 취소/무시).
- **되돌리기 없음**(이번 범위): 확인 다이얼로그가 유일한 안전장치. (전역 undo는 화면 3 '되돌리기' P1 후보와 함께 추후.)

**⚠️ 평가자 체크포인트 (초기화)**
- 버퍼 clear는 **`memset`/전체 재할당 1회**로 끝내고, 매 프레임 도는 경로에 들어가지 않게.
- 삭제 후 **자동저장이 빈 작업물을 되살리지 않는지**(레이스) 확인 — 디바운스 타이머/`saver.flush()`와 충돌 주의.
- 다이얼로그 띄운 채 뒤로가기/연타 시 **중복 삭제·dangling 상태** 가드.

### 사용자 흐름
```
도안 탭 → 캔버스 진입(이전 진행 복원) → 색칠 → 뒤로 → 자동 저장 → 갤러리 '진행 중' 갱신
                       └─ (레일 아래) 휴지통 탭 → 확인 다이얼로그 → [초기화] → 색칠 비움 + 작업물 삭제 → 캔버스 유지
```

### 플랫폼
- **iPad**: PencilKit + Apple Pencil(필압/기울기) + 손가락. 펜 전용/손가락 허용 정책 결정 필요(열린 질문).
- **Mac**: PencilKit의 macOS 지원 사용 또는 대체. **⚠️ PencilKit 네이티브 macOS 동작·가용성 확인 필요(기술 리스크)** — 안 되면 `#if os(iOS)` 분기 + Mac은 단순 `Canvas` 브러시 폴백.

### 화면 전환 / 애니메이션 — P1
- 셀 → 캔버스: 도안이 그리드 칸에서 **확대되며 열림**(`matchedGeometryEffect` 검토), 뒤로 시 역방향.

### 범위 밖 (v1 제외)
- 탭-채우기/버킷(flood fill), 스티커·패턴 브러시, 다중 레이어, 줌/팬 확대.
- 내보내기/공유(P1).
- 되돌리기/다시: v1 비선택. **단 PencilKit는 `UndoManager`로 거의 무료** → P1 최우선 후보(권장).

### ⚠️ 평가자 체크포인트 (성능)
- 매 스트로크마다 오버레이 합성·썸네일 렌더 돌지 않게(저장 디바운스).
- undo 스택이 전체 비트맵 복제하지 않게(PencilKit는 벡터 스트로크라 유리).
- 풀해상 라인아트 합성·인코딩은 백그라운드.
