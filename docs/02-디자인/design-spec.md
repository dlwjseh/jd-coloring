## 1. 디자인 컨셉 / 무드
- **키워드**: 밝고 둥글고 친근한, 크레용/파스텔 컬러링. 아이도 편하게.
- 모든 요소는 **둥근 모서리**(원·라운드 사각형), 부드러운 그림자, 따뜻한 크림 배경.
- 과하지 않게: 배경은 차분한 크림 그라데이션, 색은 프로필·버튼 등 포인트에만.

## 2. 컬러 시스템 (`palette.svg`)
| 역할 | HEX |
|------|-----|
| 배경 (그라데이션 상/하) | `#FFFFFF` → `#FFF4EC` (거의 화이트, 따뜻한 기 살짝) |
| 잉크(주요 텍스트) | `#3B3A4E` |
| 보조 텍스트 | `#9A8F86` / 옅게 `#B6A89B` |
| 카드 | `#FFFFFF`, 보더 `#EBDFD2` |
| CTA(주요 버튼)·코랄 | `#FF7A59` |
| 위험(삭제) | `#FF5A5F` |
| 프로필 링 6색 | 코랄 `#FF7A59` · 퍼플 `#8A6CFF` · 티일 `#36C5C0` · 옐로 `#FFC740` · 핑크 `#FF6FAF` · 그린 `#5FD08A` |
| 프로필 옅은 채움 | 각 링색의 파스텔 틴트 (예 코랄 `#FFE4DB`) |

프로필 링 색은 **추가된 순서대로 6색을 순환** 배정(사진이 없거나 단색일 때도 구분되게).

## 3. 타이포그래피
- 서체: **SF Pro Rounded**(시스템 라운드). 없으면 시스템 기본.
- 타이틀 800 / 48pt · 서브타이틀 24pt · 이름 라벨 700 / 27pt · 본문·버튼 21~22pt.

## 4. 컴포넌트
- **프로필 원(ProfileCircle)**: 지름 152(=r76), 링 8pt(색), 옅은 틴트 채움, 사진은 원형 마스크. 아래 이름 라벨. 그림자 `dy7/blur11`. 사진 없으면 기본 스마일 일러스트.
- **방울 배경(BubbleBackground)**: 배경 그라데이션 위에 6색 팔레트의 파스텔 방울(원)을 낮은 불투명도(0.09~0.14)로 화면 곳곳에 흩뿌림. 정적·히트테스트 비활성. 화면 크기에 비례 배치(창 리사이즈 대응). *(구현 시 누락됐다가 추가됨 — 모든 화면 배경의 기본 요소로 유지)*
- **추가 버튼(AddButton)**: 우하단 코너 플로팅 원형, 지름 96, 코랄, 흰 `＋`. 아래 "프로필 추가" 캡션.
  - *대안*: 프로필 줄 끝의 점선 `＋` 원(인라인). 코너안을 1차 추천.
- **수정 폼 카드**: 흰 카드 rx36, 내부 = 점선 원형 이미지 픽커("＋ 갤러리에서 사진 선택") + 이름 입력 필드(rx18) + [취소(아웃라인)]/[저장(코랄)].
- **컨텍스트 메뉴 팝오버**: 흰 카드, `✏️ 수정` / `🗑️ 삭제(빨강)`.
- **확인 다이얼로그**: 흰 카드, 아이콘+제목+보조문구 + [취소]/[삭제(빨강)].

## 5. 화면별 레이아웃

### 메인 — 사용자 선택 (`01-user-select.svg`)
- 상단 중앙 타이틀 "오늘은 누가 색칠할까요?" + 서브 "프로필을 선택하세요".
- 중앙: 프로필 원들이 가로 한 줄, 수평 스크롤. 오른쪽 끝 살짝 잘린 프로필로 **더 있음(스크롤)** 암시 + "‹ 좌우로 드래그 ›" 힌트.
- 우하단: 추가 버튼.

### 추가·수정 / 첫 실행 (`02-add-edit.svg`)
- 중앙 폼 카드. 좌·우 가장자리에 흩어진 프로필(A2 종료 상태)이 보임.
- **첫 실행(사용자 0명)** 에는 이 화면이 곧바로 등장. **수정**은 같은 화면을 재사용(기존 값 채워진 상태).

### 롱프레스·삭제 (`03-longpress-delete.svg`)
- ① 롱프레스 → 대상 프로필이 살짝 떠오르고(scale↑) 배경 딤 + 메뉴 팝오버.
- ② [삭제] → 확인 다이얼로그. [삭제]는 빨강.

## 6. 애니메이션 (개발자 인계용)

### A1. 진입 — 순차 슬라이드 인 (`04-anim-A1-entry.svg`)
- 각 프로필이 **화면 오른쪽 바깥 → 제자리**로, **오른쪽→왼쪽** 이동 + 페이드 인.
- **stagger**: index 기반 50~80ms 지연으로 하나씩.
- 정착 시 가벼운 스프링(예: `spring(response:0.45, dampingFraction:0.75)`).
- 트리거: 이 화면 진입 시마다(`onAppear`).

### A2. 흩어짐 + 폼 등장 (`05-anim-A2-scatter.svg`)
- 1단계: 프로필이 좌/우로 갈라져 화면 밖 + 페이드 아웃(가운데 비움).
- 2단계: 폼 카드가 **아래에서 위로** 스윽(+페이드 인). 1·2단계 살짝 오버랩.
- 역방향: 저장/취소 시 폼 내려가 사라지고 프로필 다시 모임.
- 방향 결정: 폼은 **아래→위**로 제안(키보드와 충돌 적고 자연스러움).

> ⚠️ 구현 시: stagger는 타이머 남발 대신 transition+delay로. "매 진입 재생"이라 가볍게 유지 (→ 개발평가자 체크포인트).

## 7. 접근성 / 반응형
- 터치 타깃 ≥ 44pt. 텍스트·배경 대비 확보(잉크/크림).
- **iPad**: 가로/세로 모두. 세로는 프로필 크기 유지하고 줄 길이만 조정.
- **Mac**: 롱프레스 → **우클릭(컨트롤+클릭) 컨텍스트 메뉴**로 대체. 수평 스크롤은 트랙패드 제스처. 창 리사이즈 시 프로필 줄 중앙 정렬 유지.

## 8. 디자이너가 둔 결정 / 열린 질문
- 추가 버튼: **우하단 코너** 1차 추천(인라인 `＋`는 대안).
- 프로필 사진 없을 때: 기본 **스마일 일러스트**(이번엔 단순화). 추후 동물 아바타 세트 검토 가능.
- 폼 등장 방향: **아래→위** 제안.

# 화면 2 — 색칠 도안 갤러리

입력 기획서: `docs/01-기획/requirements.md`(화면 2 섹션). 화면 1과 **동일한 배경/그림자/서체/방울**을 그대로 계승.

## 9. 레이아웃 (`06-gallery.svg`)
- **상단 바**: 좌측 **뒤로 버튼**(흰 원 + ‹). 중앙 타이틀 "무엇을 색칠해볼까요?" + 서브 "○○의 색칠 도안 · N개". 우측 **현재 프로필 칩**(작은 원 + 이름, 화면 1 링색 유지).
- **그리드**: 적응형 카드 그리드. 목업 기준 4열(폭에 따라 자동), 카드 **210×210 rx28**, 흰 배경 + 보더 `#EBDFD2` + 카드 그림자(`dy6/blur10`, opacity 0.28), 간격 60.
- **셀 내용**: 도안 일러스트(라인아트) 중앙. 카드 하단 **이름 라벨 영역(고정 높이 30pt)**:
  - 이름이 **있으면**: 텍스트 표시. 최대 1줄 말줄임(`lineLimit(1)`), 중앙 정렬, 14pt/600.
  - 이름이 **없으면**: 텍스트 없음. 영역(30pt)은 그대로 유지 → **카드 높이 통일** (이름 유무에 따라 그리드가 들쭉날쭉해지지 않게).
  - **진행 중**(현재 프로필이 색칠한 도안): 일러스트가 **파스텔로 채워짐** + 좌상단 코랄 "진행 중" 알약 배지. 이름 라벨 잉크색 `#3B3A4E`.
  - **미착수**(빈 도안): 일러스트 **잉크 외곽선만**(채움 없음). 이름 라벨 보조색 `#9A8F86`.
- **추가 버튼**: 화면 1과 **완전히 동일**한 우하단 코랄 플로팅 원(＋) + "도안 추가" 캡션.

## 10. 빈 상태 (`07-gallery-empty.svg`)
- 중앙에 점선 캔버스 + 크레용 일러스트. "아직 색칠할 도안이 없어요" + "오른쪽 아래 ＋ 버튼으로 사진을 올려 도안을 만들어요".
- 추가 버튼을 향하는 **점선 가이드 화살표**(코랄)로 첫 행동 유도. (첫 진입·0개일 때만)

## 11. 도안 업로드 (`08-upload.svg`)
- 딤 처리된 배경 위 **시트 카드**(흰색 rx40). 타이틀 "도안 만들기" + 설명 "사진을 골라 색칠 도안으로 추가해요".
- **단일 미리보기** 한 장(흰 박스 rx28, 선택한 이미지 그대로). 그 아래 **"사진 다시 선택"**(코랄 링크).
- 미리보기 아래 **이름 입력 필드**: 전체 폭, rx14, 흰 배경 + 보더 `#EBDFD2`. placeholder "도안 이름 (선택)" 보조색. 포커스 시 보더 `#FF7A59` 2pt. 입력은 선택 사항 — 비워도 저장 가능.
- 하단 [취소(아웃라인)] / [도안 저장(코랄)].
- ~~라인아트 변환·before/after·"원본 그대로" 토글~~ → **제거(2026-06-01)**. 사진을 가공 없이 그대로 도안으로 등록(사용자가 색칠 도안 이미지를 올린다고 가정).

## 12. 화면 2 애니메이션 — G1 화면 전환

기획서 `requirements.md` 화면 2 §「G1」 구체화. **프로필 화면 요소가 빠지고 → 갤러리 도안이 밑에서 우수수 올라오는** 전환. 스토리보드 `16-anim-G1-slide.svg`.

> 💡 **설계 변경 이력**: 처음엔 "탭한 프로필이 펼쳐지는" 네이티브 zoom(`.navigationTransition(.zoom)`)으로 잡았으나(스토리보드 `15`, 폐기), 펼침 느낌이 어색해 **흩어짐 + 밑에서 올라옴** 방식으로 변경(2026-06-02). 배포 타깃 **iOS 18 / macOS 15 상향은 유지**(다른 최신 API 여지).

### 12-1. 전환 메커니즘
- **화면 전환 자체는 애니메이션 없이**(`Transaction.disablesAnimations`로 push/pop) 처리하고, 연출은 **양쪽 화면의 콘텐츠**가 담당. 두 화면이 **같은 크림 배경**을 공유하므로 즉시 전환이 눈에 띄지 않아, 한쪽이 빠지고 다른 쪽이 들어오는 흐름이 매끄럽게 이어진다. (기본 iOS 가로 슬라이드 푸시를 쓰지 않는 이유)

### 12-2. 정방향 (프로필 → 갤러리)
**① 프로필 화면 빠짐** (`easeIn`, 0.30s):
- 상단 타이틀 → **위로**(offset y −260) + 페이드
- 프로필 원들 → **좌·우로 흩어짐**(A2 흩어짐과 동일, 가운데 기준 양옆 화면 밖) + 페이드
- 우하단 '도안 추가' 버튼 → **아래로**(offset y +260) + 페이드, 드래그 힌트도 아래로

**② 갤러리 등장** (push 직후 `onAppear` 트리거):
- 상단 타이틀 → **위에서 내려오며**(offset y −30→0) + 페이드, spring(0.5/0.85)
- 도안 카드 → **밑에서 우수수 올라옴**(offset y +56→0) + 페이드, **index 기반 stagger**(0.04s 간격, 14개에서 포화), spring(0.5/0.82) — `GridEntrance`
- 우하단 '도안 추가' 버튼 → **밑에서 올라옴**(offset y +90→0), 0.12s 지연

### 12-3. 역방향 (갤러리 → 프로필) — 정방향의 대칭
**① 갤러리 빠짐** (`easeIn`, 0.30s): 타이틀 **위로**, 카드·추가 버튼 **아래로** 빠짐(진입의 역).
**② 프로필 복귀** (spring 0.5/0.85): 흩어졌던 프로필이 **좌·우에서 제자리로**, 타이틀 **위에서 내려옴**, 버튼 **아래에서 올라옴**(흩어짐의 역). **A1(오른쪽 슬라이드)는 재생하지 않음** — 흩어짐의 되돌림으로 통일.

### 12-4. 구현 메모 / 견고성 (개발평가자 검수 반영 2026-06-02)
- **stagger**: 타이머 금지 — `.animation(...).delay(index*간격)`로 구현(`GridEntrance`/`StaggeredEntrance`).
- **전환 지연**: 빠짐 연출(0.30s) 종료 후 push/pop. `Task.sleep`(취소 가능) 사용, `asyncAfter` 금지. duration은 단일 상수로 애니메이션과 통합.
- **연타/재진입 가드**: 진입 시 `transitioning`(프로필) / `guard !exiting`(갤러리)로 **중복 push·dangling dismiss 차단**. (아이 사용자 더블탭 대비)
- **dangling 방지**: `GalleryView.onDisappear`에서 전환 Task 취소 → 외부 요인 pop 시 늦은 dismiss 방지.
- **접근성 — Reduce Motion**: "동작 줄이기" ON 시 **이동·stagger 생략하고 페이드만**, 지연 0.18s로 단축.
> ⚠️ **개발평가자 체크포인트**: "매 진입 재생"이라 무거우면 즉시 체감 — 도안 多일 때 카드 동시 모션 60fps 유지(lazy 렌더 + 가벼운 offset/opacity로 양호 확인).

### 18-2. 색연필 질감 표현 (`14-pencil-texture.svg`)
> 목표: **반투명 쌓임 + 종이결, 은은하게.** 마커처럼 평평하게 덮지 않고, 칠할수록 농담이 쌓이고 종이 결이 살짝 비친다.

- **종이결(tooth)**: SVG 시안은 `feTurbulence`(fractalNoise, `baseFrequency≈0.5`, 2 octave) 기반 미세 톱니를 **도형 알파 안에서만** `feComposite operator="out"`으로 살짝 덜어내 흰 종이가 비치게 표현(`filter id="pencil"`). 변조 폭은 작게 — 멀리선 거의 채워 보이고 가까이서 결이 느껴지는 정도. **결은 종이(캔버스 좌표)에 고정** — 칠을 움직여도 결이 떨리지 않음.
- **쌓임(build-up)**: 시안에선 **불투명도 누적**으로 시연(1획 0.4 → 2획 0.65 → 3획 0.88). 같은 색의 진한 값에서 멈춤. 윗 꽃잎(2패스, 진한 핑크 `#E79BB7`)과 옆 꽃잎(1패스, 옅은 `#F7C8D8`)의 농담 차이로 캔버스 시안에 반영.
- **한 획은 균일 / 겹획은 진해짐**: `14`의 ③ 다이어그램 — 한 드래그 안에선 자기 겹침에도 얼룩 없이 균일, 다른 획과 교차한 곳만 진해짐.
- **실제 구현 주의**: SVG 필터는 **느낌 전달용 시안**일 뿐. 결 강도/입자 크기·쌓임 곡선은 `/개발` 단계 실제 래스터 렌더에서 미세조정(SVG `feTurbulence` 값과 1:1 대응 아님). 엔진은 **반투명 source-over 누적 + 고정 그레인 마스크 + 획 내 1회 반영(커버리지 마스크)** 으로 구현.

### 18-3. 디자이너 결정 / 메모
- 도구 라벨 표기: 아이콘 + 섹션 라벨 "도구"로 처리(개별 "마커/색연필" 텍스트는 39pt 칸에 안 들어가 생략). 명칭 확정은 개발 재량.
- **기본 선택 도구**: 하위 호환상 기존 **마커가 기본**(앱 첫 진입). 시안은 색연필 선택 상태를 보여주기 위해 색연필 활성으로 표시.
- **필압·기울기(Apple Pencil)**: 이번 범위 밖(P1). `14` 우하단에 P1 후보로 명시.

## 27. 캔버스 줌/팬 (`27-canvas-zoom-pan.svg`) — 추가 2026-06-04

입력 기획서: `requirements.md` 화면 3 §「캔버스 줌/팬」. 두 손가락 핀치로 도안을 최대 3배까지 확대하고, 확대 상태에서 패닝하며 색칠, 더블탭으로 즉시 원위치 복귀.

### 27-1. Frame A — 줌인 상태 (2×)

**배율 뱃지**
- **노출 조건**: `scale > 1.0` 일 때만 표시. `scale == 1.0` 이면 완전 숨김(opacity 0, 공간 차지 없음).
- **위치**: 캔버스 카드 좌상단 코너 안쪽. 뒤로 버튼/도안명과 겹치지 않게 캔버스 rect 기준 `x+8, y+8`.
- **스펙**:

| 항목 | 값 |
|------|-----|
| 배경 | `#3B3A4E` opacity 82% |
| 크기 | `64×30`, `rx15` |
| 텍스트 | `"Nx"` (N = 반올림 배율) · 흰색 · `15pt/800` |
| 등장/소멸 | `withAnimation(.easeInOut(duration:0.2))` |

**캔버스 영역 동작**
- `scaleEffect` + `offset` 변환은 **캔버스 카드 내부 콘텐츠**에만 적용. 카드 테두리·레일·저장 버튼·초기화 버튼은 고정.
- 도안 비율은 그대로 유지하며 확대. 줌 중심 = 두 손가락 midpoint.

### 27-2. Frame B — 제스처 가이드

#### 핀치 줌
- `MagnificationGesture`(SwiftUI) 또는 `UIPinchGestureRecognizer`(UIKit 래퍼).
- `scaleState = clamp(currentScale * gestureScale, 1.0, 3.0)`.
- `onEnded`에서 경계 이하면 스프링 복귀(`withAnimation(.spring())`).

#### 패닝 (두 손가락 드래그)
- 한 손가락 드래그(색칠)와 분리: `UIGestureRecognizerDelegate.numberOfTouches == 2` 일 때만 패닝 처리.
- 오프셋 클램핑: `maxOffset = canvasSize * (scaleState - 1) / 2`. 도안 가장자리가 뷰 영역 밖으로 완전히 나가지 않음.
- `onEnded`에서 경계 초과 시 스프링 복귀.

#### 더블탭 리셋
- `TapGesture(count: 2)` + `numberOfTouches == 2` 조건 (두 손가락).
- `withAnimation(.spring(response: 0.42, dampingFraction: 0.78)) { scaleState = 1.0; offsetState = .zero }`.
- 한 손가락 더블탭은 색칠 제스처와 충돌 → **두 손가락으로만** 지정.

### 27-3. 색칠 좌표 역변환 (⚠️ 핵심)
줌인 상태에서 색칠 터치 포인트를 캔버스 로컬 좌표로 변환해야 함:
```
canvasPos = (touchPoint − offsetState) ÷ scaleState
```
이 역변환 없이는 확대 상태에서 색칠이 엉뚱한 위치에 찍힘.

### 27-4. penOnly 설정과 상호작용
- `penOnly == true` 여도 **핀치·패닝·더블탭은 손가락 허용**. 손가락으로 색칠만 차단.
- 드래그 제스처 `onChanged` 진입 시점에 `UITouch.type` 확인 → `.pencil` 이 아닌 `.direct` 이면 색칠 경로만 early return.

### 27-5. 접근성
- 핀치/패닝은 시스템 표준 제스처라 별도 `accessibilityActivate` 불필요.
- 더블탭 리셋: `accessibilityCustomAction(name: "원래 크기로")` 추가 권장(두 손가락 더블탭이 일부 VoiceOver 제스처와 충돌 가능).

### 27-6. 범위 밖
- 1× 미만 축소, 한 손가락 더블탭 리셋, 줌 +/- 레일 버튼, 줌 상태 저장, 첫 진입 더블탭 힌트 — 모두 이번 제외.

### 30-1. 업로드 시트에 '앨범' 줄 추가 (Frame ①)
- 기존 구조(미리보기 + "사진 다시 선택" + 이름 필드 + 버튼) 그대로, **이름 필드 아래 '앨범' 선택 줄** 삽입.
- **줄 구성**: 좌측 라벨 "앨범" `16pt/700/#3B3A4E` + 우측 **셀렉터 알약**(`304×46 rx12`). 셀렉터 = 현재 선택 앨범의 작은 컬러 칩 + 이름 + 우측 `▾`.
- **기본값 = 지금 들어와 있는 앨범**(미분류에서 추가했으면 "미분류"). 그 아래 힌트 "기본값 = 지금 들어와 있는 앨범 · 탭하면 변경 ▸" `13pt/#B6A89B`.
- [도안 저장]은 **항상 활성**(앨범은 기본값이 늘 있으므로).

### 30-2. 앨범 고르기 펼침 (Frame ②)
- 셀렉터 탭 → 셀렉터 보더 코랄 강조 + **드롭다운 리스트**(`304 너비 rx16`, 흰 카드 + 섀도) 등장.
- **리스트 항목**: 사용자 앨범(생성순) → **미분류**(`#9A8F86`) → 구분선 → **＋ 새 앨범 만들기**(코랄).
- **현재 선택** 항목 = 옅은 코랄 틴트 배경 `#FFF1EC` + 코랄 텍스트 + 우측 체크. 항목 높이 ~42pt(터치 44pt+ 확보).
- **＋ 새 앨범 만들기** 탭 → §29 만들기 시트(P1, 즉석 생성 후 그 앨범으로 선택).
- 색 선택/항목 탭 → 드롭다운 닫힘 + 셀렉터에 반영.

### 30-3. 롱프레스 [앨범 이동] (갤러리, §13 갱신)
- 갤러리 도안 롱프레스 메뉴가 **4항목**으로: ① 이름 수정 ② **앨범 이동** ③ 내 색칠 초기화 ④ 도안 삭제(빨강).
- [앨범 이동] 탭 → §30-2와 동일한 **앨범 고르기 시트/드롭다운** 재사용 → 고른 앨범으로 `Template.categoryId` 갱신(미분류 ↔ 앨범 양방향).

### 30-4. 접근성
- 셀렉터 `accessibilityLabel("앨범")` + `accessibilityValue(현재 앨범명)`, `accessibilityHint("탭하면 앨범을 바꿔요")`.
- 드롭다운 각 항목 버튼, 현재 선택 `.accessibilityAddTraits(.isSelected)`.
